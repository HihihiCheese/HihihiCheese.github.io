Hicheese Graphic Designer Tools 1.0

发表于2026-03-06|更新于2026-03-06|Dev Log
|浏览量:|评论数:

1. 背景

在学院新媒体部门的日常工作中，公众号推文美编排版是一项重复性极高的任务：从 Word 稿件中提取文字和图片，按照固定视觉规范在秀米等编辑器中手动拼装模板，最终发布到微信公众号。

一篇典型活动新闻稿，美编排版通常需要 30-60 分钟。但排版规范本身是高度结构化的——标题、正文、图片、署名的位置和样式都有明确约定。整个流程实际具备自动化的可能性。

Hicheese Graphic Designer Tools 1.0 目标：将 Word 稿件到公众号推文的排版流程自动化

项目 GitHub 主页：HihihiCheese/Graphic_Designer_Tools

2. 系统架构

整个工具采用三阶段 Pipeline 设计：

  • tool1_extract：解析 Word 文档，提取文字段落、图片，调用 LLM 进行智能结构化
  • tool2_layout：读取结构化 JSON，选择模板，渲染为微信公众号兼容的 HTML
  • 发布阶段：目前为手动复制粘贴至公众号后台，Phase 3 的 API 推送尚在规划中（需要找管理员老师获得公众号开发者权限）

2.1 Phase 1: 内容提取 (tool1_extract)

核心模块 docx_parser.py 基于 python-docx 库，逐段落遍历 Word 文档的 XML 结构，将内容分类为三种类型：

  • 纯文字段落：提取文本内容和 Word 样式名称
  • 纯图片段落：从 a:blip 元素中提取嵌入图片，按序编号保存
  • 混合段落：文字与图片共存，拆分为独立内容块

提取结果以 raw_paragraphs.json 形式保存，随后交给 LLM 处理模块 llm_processor.py 进行智能结构化：

  • 确认/修正标题
  • 生成摘要
  • 判断文章类型 (news / interview / lecture)
  • 识别小节标题 vs 正文段落
  • 提取署名信息 (文字、图片、美编、责编、审核)
  • 标记需要强调的关键词

LLM 调用支持 Claude 和 Gemini 两种 Provider，通过 CLI 的 --print 模式以 subprocess 方式调用。当 LLM 不可用时，系统会自动回退到基于规则的处理逻辑——通过段落长度、关键词匹配、位置特征等启发式规则完成结构化。

2.2 Phase 2: 排版生成 (tool2_layout)

排版引擎基于 Jinja2 模板，将 extracted.json 渲染为微信公众号兼容的 HTML。核心设计：

  • 复用秀米 HTML 片段：不从零写 CSS，直接从秀米模板中拆解可复用组件 (text_card、image_frame、decorator_line 等)，保留原始内联样式
  • 全量 CSS 内联：微信公众号后台会删除 <style> 标签，因此所有样式必须写成 inline style
  • 智能图片样式：连续 3 张及以上图片才启用偏移轮换样式，单独图片一律居中

模板体系

目前已实现 8 套模板，覆盖学院常见推文类型：

模板名 风格 适用场景
news 蓝色条纹卡片 通用新闻稿 (默认)
news_blue2 深蓝边框 + 折角装饰 较短的活动稿
news_red 红色双线 + 丝带图片 党建活动 (复杂版)
news_red2 红色边框 + 角落装饰 党建活动 (简约版)
news_red3 红色卡片 + 半圆装饰 党建活动 (变体)
news_cyan 青蓝渐变 + 橙色徽章 学术论坛/学院动态
news_purple 紫色渐变 + 方块装饰 正式通知/会议
lecture 蓝线 + 虚线框 讲座/报告会

组件化架构

components.py 中封装了约 35 个 HTML 组件函数，按模板风格分组：

  • 通用组件：head_image、tail_image
  • 蓝色系：text_card、image_frame、heading_block、credits_block
  • 红色系：red_text_card、red_image_with_ribbon、red_numbered_heading
  • 青色/紫色：cyan_text_card、purple_heading
  • 讲座专用：lecture_poster、lecture_info、lecture_card

每个组件函数返回一段完整的内联样式 HTML 片段，模板通过 Jinja2 的 globals 机制直接调用。

3. 技术栈

层级 技术
文档解析 python-docx (XML 级操作)
智能处理 Claude CLI / Gemini CLI (subprocess)
模板渲染 Jinja2
图片处理 Pillow
输出格式 微信公众号兼容 HTML (全内联 CSS)

4. 实际效果与验证

该工具已在学院新媒体部门的实际推文产出中得到验证。自动化排版的推文已成功发布至公众号，效果可参见：

  • 公众号推文示例

实际使用上，工具将单篇推文的排版时间从 30-60 分钟压缩至 5 分钟以内（主要是人工审校和微调的时间），排版质量与手工制作基本一致。

5. 已知局限与后续计划

  • --no-llm 模式对讲座类型的识别能力有限，"论坛"等关键词尚未覆盖
  • 讲座类型的 extracted.json 疑似需要手动调整为 lectures 列表格式，原因未知
  • 模板自动推荐功能 (根据 category 和关键词匹配最佳模板) 尚未实现，暂时需要手动指定
  • Phase 3 的微信公众号 API 自动推送功能尚在规划中（尚未联系管理员老师获得开发权限）
