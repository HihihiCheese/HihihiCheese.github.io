Learning-Assisted Relaxations and Approximations

发表于2026-02-10|更新于2026-02-11|Academic Notes
|浏览量:|评论数:

1. 引言

同一个固定的物理系统，在不同外部参数（Parameters, θ ）下存在重复求解的问题。

x∗(θ)=arg⁡minxf(x;θ)s.t. g(x;θ)≤0

传统的优化算法（如内点法、单纯形法）每次求解都视作一次 Cold Start，忽略历史数据可能蕴含的规律。

Learning-Assisted Optimization ( Relaxations and Approximations ) 的核心在于：利用历史数据训练 ML 模型，预测优化的关键信息，从而跳过繁琐的试错步骤。

2. 回顾：经典分解与迭代

在引入 ML 之前，需要先理解经典算法是如何工作的。它们的迭代过程正是 ML 试图加速的对象。

2.1 拉格朗日松弛 (Lagrangian Relaxation, LR)

拉格朗日松弛的核心思想是将"难约束" Ax=b 转移到目标函数中，通过调节乘子 λ 来惩罚违反约束的行为。

对偶更新公式：

λk+1=λk+αk⋅(Axk−b)⏟Constraint Violation

数学本质：

  • 次梯度 (Subgradient)：约束违反量 (Axk−b) 本质上是对偶函数 q(λ) 的次梯度。
  • 步长 ( αk )：通常采用 Polyak Step Size 或衰减步长，以保证收敛。

痛点：存在试错和修正的过程（约束违反则增加 λ ）。对于大规模问题，这种震荡收敛的过程非常缓慢。

2.2 优化条件分解 (Optimality Condition Decomposition, OCD)

OCD 是一种特殊的 LR 方法。与传统 LR 直接分解原问题不同，OCD 试图直接分解 KKT 条件。

核心逻辑：

  • Shadow Prices：在 OCD 中，对偶变量 λ 被直接解释为 Shadow Prices，即目标函数对约束边界变化的灵敏度 ( ∂f∂b )。
  • 去中心化 (Decentralization)：通过固定耦合变量的一份拷贝，迭代求解另一份拷贝。
  • 梯度下降视角：OCD 的迭代过程可以被视为在对偶空间上的梯度下降。

优势：OCD 不需要像 LR 那样在每次迭代都完全求解子问题，它允许子问题只进行一步更新，因此计算效率往往更高，且天然适合分布式计算（Decentralization 性质）。

3. Learning-Assisted Relaxations

将 ML 引入上述过程时，实际上是在用预测代替迭代试错过程。

3.1 Warm Starting

目标

  • 预测最优解 x∗ 的近似值 x^ ，作为求解器的初始点。

机制

  • 凸问题：提供高质量的 xinit ，显著减少内点法（IPM）的迭代次数。
  • 非凸问题：提供高质量的 xinit ，显著减少内点法（IPM）的迭代次数，同时 xinit 位于全局最优解的吸引域 (Basin of Attraction) 附近，避免陷入局部最优，提高求解质量

ML 方法

  • Regression：最基础的方法，如 MLP 或 SVR。
  • Random Forests / XGBoost：适合表格型数据。
  • Graph Neural Networks (GNN)：适合工程问题，捕捉问题可能的拓扑结构。

文献

  • Baker, K. (2019). Learning warm-start points for AC optimal power flow. IEEE MLSP. (文献中证明了简单的回归就能大幅加速问题求解)
  • Mulvey, J., et al. (2020). A machine learning framework for solving large-scale stochastic optimization problems.

3.2 Identifying Active Constraints

目标

  • 预测哪些不等式约束在最优解处会取等号（活跃约束/紧约束）（Active, g(x)=0 ）。

机制

  • 一旦识别出 Active Set A(θ) ，问题就简化为只包含等式约束的系统，维度大幅降低。

纠错

  • 如果 ML 漏掉了某个约束，通过 Constraint Generation 将其加回，或者增大惩罚系数重新求解（针对 Augmented Lagrangian Method，见 Chapter 6）。

ML 方法

  • Multi-label Classification：每个约束对应一个二分类标签（Active/Inactive）。
  • Deep Classification Networks：输入 θ ，输出 A(θ) 。

文献

  • Misra, S., et al. (2022). Learning active constraints for optimal power flow. (专门探讨如何利用分类器筛选约束的文献)
  • Deka, D., & Misra, S. (2019). Learning for DC-OPF: Classifying active sets using neural networks.

3.3 Identifying Invariant Variables

目标

  • 预测哪些变量在最优解中保持不变。

机制

  • 固定这些变量，直接降低问题维度（Model Reduction）。

ML 方法

  • Classification + Conformal Prediction (保形预测)

  • 为什么需要 Conformal Prediction？ 直接分类可能出错。Conformal Prediction 可以给出一个包含真值的置信区间（Set-valued prediction），保证以 1−ϵ 的概率（如 95%）不会错误地固定变量（导致 Suboptimal），从而保证 Optimality Preservation。

关于 Conformal Prediction

  • Conformal Prediction (CP) 是一种统计框架，用于量化机器学习模型的预测不确定性。它将模型的点预测（Point Prediction）转化为预测集合（Prediction Set）或预测区间（Prediction Interval），并提供具有统计学效力的覆盖保证。

  • CP 的标准实施步骤如下：

    1. 数据划分：将数据集划分为训练集（Training Set）和校准集（Calibration Set）。
    2. 模型训练：在训练集上训练基础模型 f^ 。
    3. 计算非一致性分数（Non-conformity Score）：在校准集上，计算每个样本的非一致性分数 si 。该分数衡量预测结果与真实标签的偏差程度。
      • 回归问题示例： si=|yi−f^(xi)| （残差绝对值）。
      • 分类问题示例： si=1−p^(yi|xi) （1 减去真实类别的预测概率）。
    4. 计算分位数（Quantile）：计算校准集分数的 (1−α) 分位数 q^ 。
    5. 构建预测集：对于新的测试样本 xnew ，其预测集合包含所有满足 s(xnew,y)≤q^ 的候选 y 。

文献

  • Vovk, V., Gammerman, A., & Shafer, G. (2022). Algorithmic Learning in a Random World. Springer. （关于 Conformal Prediction 的详细介绍）
  • Angelopoulos, A. N., Bates, S., et al. (2021). Learn then Test: Calibrating Predictive Algorithms to Achieve Risk Control. Nature. （关于为什么使用 CP 适合 Identifying Invariant Variables）

3.4 Linearizing & Convexifying

目标

  • 对于非凸项（如 xy ），预测最佳的泰勒展开参考点 (x^,y^) 。

思考：为什么需要 Linearizing & Convexifying？

  • 线性化（Linearizing） 的核心目的是为了利用成熟算力。目前的工业级求解器基于 Branch-and-Cut 算法，在处理 MILP 时具有极高的效率和确定性，能保证找到全局最优解。而非线性规划（NLP）往往难以收敛或计算极其缓慢。通过引入辅助变量或分段逼近将模型线性化，把复杂的非线性工程问题转化为标准数学模型，从而借用求解器的强大算力快速求出精确解。

  • 凸化（Convexifying） 的核心目的是为了保证解的质量（全局最优）。在非凸优化问题上的梯度下降等算法容易陷入局部最优。通过 Convex Relaxation 或构建 Convex Hull ，将凹凸不平的可行域填补为凸集，虽然可能扩大了可行域，但它能保证找到的是全局最优（或其严谨的下界），对于评估原问题解的优劣以及在分支定界等算法中确定搜索边界至关重要。

为什么需要预测？

  • 传统的线性化通常是在上一次迭代点展开。但如果初始点选得不好，线性化后的可行域可能离真实可行域相差较大。

ML 的作用

  • ML 预测的是全局最优解附近的那个点。在这个点进行泰勒展开 f(x)≈f(x^)+∇f(x^)T(x−x^) ，得到的线性平面是与原非凸曲面贴合度最高的切平面，从而使得近似误差最小。

4. Learning-Assisted Approximations

不仅用 ML 预测参数，还可以将训练好的 ML 模型直接作为约束嵌入到优化问题中。

4.1 普通神经网络的嵌入 (MILP)

当我们用神经网络拟合复杂物理过程（如 z=NN(x) ）并嵌入优化模型时，ReLU 激活函数 z=max(0,z^) 带来了非凸性。

Big-M 转化

  • 引入 0-1 整数变量 y 和大常数 M ：
z≤M⋅y,z≤z^+M(1−y),y∈{0,1}

后果

  • 问题变成了 MILP。虽然精确，但求解难度是 NP-Hard。同时网络层数加深，整数变量爆炸式增长，求解时间不可接受。

4.2 Input Convex Neural Networks (ICNN)

为了解决计算复杂度，Amos 等人 (ICML 2017) 提出了 ICNN。

结构约束

  • 非负权重：所有传递中间层信号的权重必须非负 ( Wk≥0 )。
  • 直连通道 (Passthrough)： zk+1=σ(Wkzk+Vkz0+bk) ，允许原始输入直接影响每一层。
  • 凸激活：使用 ReLU 等凸函数。

思考：为什么限制权重非负就能变线性？

  • 由于是在做 Minimization 问题，且 W≥0 保证了凸性传递。原本非线性的 ReLU z=max(z^,0) 可以被松弛为一组纯线性约束：
z≥z^,z≥0
  • 在最小化目标函数的作用下，变量 z 会自动贴合到 max(z^,0) 的边界上。

结论

  • ✅ 不需要整数变量
  • ✅ 不需要 Big-M
  • ✅ 问题变成一个简单的 LP

Bridge between data and solver

  • 利用深度学习强大的拟合能力，ICNN 可以逼近极其复杂的、非线性的物理规律（只要它是凸的）。再利用特殊的结构设计，ICNN 可以直接转化为 Linear Programming (LP)。这使得 ICNN 可以完美嵌入到 Benders Decomposition（作为子问题切平面）或 Bi-level Optimization（作为下层问题的 KKT 条件替代）中，实现大规模系统的极速求解。

文献

  • Amos, B., Xu, L., & Kolter, J. Z. (2017). Input convex neural networks. ICML. (关于 ICNN 的起源)
  • Grimm, et al. (2022). The convex relaxation barrier, revisited: Tightened single-neuron relaxations for neural network verification. (关于 NN 的凸松弛)

5. 总结

5.1 方法论总结与对比

方法 核心思想 主要特点 典型应用场景
Warm Starting 预测最优解(作为初始点) 减少求解器迭代次数；改善非凸问题的收敛区域 重复求解的非凸优化；大规模线性规划
Active Constraints 预测最优解处的活跃约束 降低问题维度；可能需配合纠错机制 约束数量远大于变量数量的系统
Invariant Variables 预测最优解中的不变变量 通过 Conformal Prediction 提供统计保证 高维稀疏优化问题
Linearizing & Convexifying 预测最佳线性化参考点 提供更紧的线性逼近；避免迭代中的错误起点 非线性非凸优化
ICNN 凸神经网络嵌入为 LP 约束 无需整数变量；保持凸性；可直接嵌入求解器 需要拟合凸物理过程的复杂优化问题
